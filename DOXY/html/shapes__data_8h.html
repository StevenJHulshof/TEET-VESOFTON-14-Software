Detailed Description

Functions for processing the shapes.

These functions are present in the logic layer of the VGA library. They will process the data input in the presentation layer.

Function Documentation

status_t VGA_L_setLineData ( sPosition_t  endPointPos[2],
color_t  lineColor,
uint8_t  lineWeight 
)

Processes line data to pixel positions.

This function is called by VGA_P_drawLine() or VGA_L_setPolygonframe() for processing. After calculating the line data using the Bresenham algorithm this function will call VGA_L_setPixelData() for draw the line into the VGA_RAM1 DMA buffer.

Parameters
endPointPosEnd point positions of the line.
lineColorColor of the line. Ranging 0 to 255.
lineWeightWeight of the line in pixels. Ranging 1 to 255.
Returns
Status of operation. Possible return values:
VGA_SUCCESS - Operation is success.
VGA_PIXEL_NOT_SET - Set pixel operation has failed.

Process polygon data to pixel position.

This function is called by VGA_P_drawPolygon() for processing and VGA_L_setPrimitiveData() for processing default primitive shapes. The polygon is set by creating the fill by calling VGA_L_setPolygonFill() and by creating the frame by calling the VGA_L_setPolygonFrame().

Parameters
verticePosPositions of the polygon vertices.
numberOfVerticesTotal number of polygon vertices. Must be the same as the array length of verticePos.
lineColorLine color of the polygon. Ranging 0 to 255.
fillColorFill color of the polygon. Ranging -1 to 255.
lineWeightWeight of the line. Ranging 1 to 255.
Returns
Status of operation. Possible return values:
VGA_SUCCESS - Operation is success.
VGA_PIXEL_NOT_SET - Set pixel operation has failed.

Fills polygon with fillColor parameter.

This function is called by VGA_L_setPolygonData() for processing the polygon fill. It uses the efficient version of the Point-In-Polygon algorithm to calculate the fill pixel positions. After calculations it will call the VGA_L_setPixelData() for processing.

Parameters
verticePosPositions of the polygon vertices.
numberOfVerticesTotal number of polygon vertices. Must be the same as the array length of verticePos.
imageBorderImage border constraints. These consist of the minimum x,y coordinates and the maximum x,y coordinates.
fillColorFill color of the polygon. Ranging 0 to 255.
Returns
Status of operation. Possible return values:
VGA_SUCCESS - Operation is success.
VGA_PIXEL_NOT_SET - Set pixel operation has failed.

Process primitive data to pixel position.

This function is called by VGA_P_drawPrimitive() for processing. The primitive is calculated using an ellipse calculation. The different shapes are created using different sample sizes during the ellipse calculation. After the calculations, this function will call VGA_L_setPolygonData() to set the primitive shape in the VGA_RAM1 DMA buffer.

Parameters
centerPointPosCenter point position of the primitive. Contains x and y coordinates.
radiiRadius x and y of the primitive. Must be above 0.
rotationDegreesRotation of the primitive in degrees.
lineColorPrimitive frame line color. Ranging 0 to 255.
fillColorPrimitive fill color. Ranging -1 to 255.
lineWeightPrimitive frame line weight. Ranging 1 to 255.
primitiveShapePrimitive default shape as the primitiveShape_t enum defines.
Returns
Status of operation. Possible return values:
VGA_SUCCESS - Operation is success.
VGA_PIXEL_NOT_SET - Set pixel operation has failed.
